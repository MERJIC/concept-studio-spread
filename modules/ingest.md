# Concept Ingest — 概念输入管道

## 使用方式

```
/concept-ingest https://example.com/article
/concept-ingest [粘贴一段文字]
```

## 执行步骤

### Step 1 — 获取内容

**URL 输入**：用 `mcp__web-search__web_search` 搜索 URL 对应的内容，或请用户提供页面正文。
**文本输入**：直接使用用户粘贴的内容，无需额外处理。

### Step 2 — 概念提取

从内容中识别值得沉淀到概念库的候选概念。

**纳入标准（需同时满足）：**
- 有明确的学科命名（不是泛泛话题）
- 有特定的理论机制，不只是常识描述
- 来自某个学科传统（哲学 / 心理学 / 经济学 / 社会学 / 传播学 / 管理学 / 生物学 / 物理学 / 人类学 / 政治学 / 艺术）
- 对跨领域思考有迁移价值

**排除：**
- 日常词汇（即使文章反复使用）
- 过于宽泛的话题（"数字化""创新"不是概念）
- 专有名词（人名、产品名、公司名）
- 纯技术术语（API、算法名称等）

提取 1-5 个候选概念，按沉淀价值从高到低排序。

### Step 3 — 去重检查（轻量索引）

读取 `{ROOT}/memory/concept_lite.json`，对每个候选概念执行去重：

1. **精确匹配**：检查 `候选中文名 in names` → 存在则标注「已有」
2. **别名匹配**：检查 `name_aliases[候选名]` → 发现则标注「可能重复：与 {别名} 同义」
3. **英文名匹配**：检查 `name_en_index[候选英文名.lower()]` → 存在则标注「英文名已有：{对应中文名}`

> **首次使用？** 如果 `{ROOT}/memory/concept_lite.json` 不存在，跳过去重步骤，直接进入展示候选。

### Step 4 — 展示候选，等待确认

```
从「{来源标题或前30字}」发现 N 个候选概念：

1. 概念名（English Name）— 一句话说明沉淀价值
2. 概念名（English Name）— ...
3. [已有] 概念名 — 库中已有页面

建哪几个？（全部 / 编号，如 1,3）
```

**不得在等待回复前自行执行写入。**

### Step 5 — 生成概念页

对用户选中的每个概念，依次执行：

1. 用 `date` 命令获取当前日期
2. **读取 `{ROOT}/memory/concept_relations.md`**：查看该概念的集群归属和孤立概念列表
   - 若属于某集群，写正文时在自然出现处用 `[[]]` 链接集群内相关概念
   - 若出现在孤立列表，说明它此前无出链，建完页后它就有了入链
3. 按下方规范生成概念页内容
4. 用 Write 工具写入 `{ROOT}/概念页/{中文名}.md`
5. 写完后立即执行自检（见「写完后必须自检」）
6. 运行 `python3 {ROOT}/scripts/build_index.py --root {ROOT} --incremental` 刷新索引
   - 如果脚本不存在，提示用户索引需要手动刷新或稍后配置脚本

多个概念逐个写入，每个写完后输出确认，再写下一个。

### Step 6 — 可选：发起圆桌

所有概念页写入完成后，从本次新建的概念中找出张力最强的一个，询问是否发起圆桌讨论：

> 要不要就「{概念名}」发起圆桌讨论？我想到一个问题：
> **{问题}**
> 或者你来出题。

问题生成规则：必须是开放性问题，指向这个原理的核心张力或实践困境，不超过 2 句话。

用户确认后读取 `modules/roundtable.md`，以该问题为议题执行圆桌流程；否定则流程结束。
