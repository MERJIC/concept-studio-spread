# Concept Ingest — 概念输入管道

## 使用方式

```
/concept-ingest https://example.com/article
/concept-ingest [粘贴一段文字]
```

## 执行步骤

### Step 1 — 获取内容

**URL 输入**：使用当前 Agent 可用的网页读取或搜索能力获取 URL 内容；如果无法访问，再请用户提供页面正文。不要绑定某个平台的专用工具名称。
**文本输入**：直接使用用户粘贴的内容，无需额外处理。

### Step 2 — 概念提取

从内容中识别值得沉淀到概念库的候选概念。

**纳入标准（需同时满足）：**
- 有明确的学科命名（不是泛泛话题）
- 有特定的理论机制，不只是常识描述
- 来自某个学科传统（哲学 / 心理学 / 经济学 / 社会学 / 传播学 / 管理学 / 生物学 / 物理学 / 人类学 / 政治学 / 艺术）
- 对跨领域思考有迁移价值

**排除：**
- 日常词汇（即使文章反复使用）
- 过于宽泛的话题（"数字化""创新"不是概念）
- 专有名词（人名、产品名、公司名）
- 纯技术术语（API、算法名称等）

提取 1-5 个候选概念，按沉淀价值从高到低排序。

### Step 3 — 去重检查

**一次性批量查重**（一条命令，所有候选一起交）：

```bash
python3 scripts/sync_db.py -d "候选1名" "英文名1" "候选2名" "英文名2" ...
```

输出三种结果：
- **✅ 可用** — 无冲突，可以建页
- **⚠ 弱命中** — 子串/关键词重叠，标注为「可能重复」
- **❌ 重复** — 精确/别名命中，标注为「已有」

不再读取 concept_lite.json、INDEX.md 或任何索引文件做手动比对。

### Step 4 — 展示候选，等待确认

```
从「{来源标题或前30字}」发现 N 个候选概念：

1. 概念名（English Name）— 一句话说明沉淀价值
2. 概念名（English Name）— ...
3. [已有] 概念名 — 库中已有页面

建哪几个？（全部 / 编号，如 1,3）
```

**不得在等待回复前自行执行写入。**

### Step 5 — 生成概念页

对用户选中的每个概念，依次执行：

1. **准备入口场景**：
   - `source: 概念跳跃` 或从 hop 沉淀路由进来时：读取 `modules/parable.md` → Step 2 — 构建故事，按寓言同一流程写完整故事体（约 300–500 字）。不得写成 2–5 句梗概。与寓言的唯一差别：入口场景不必隐藏概念名；仍须动作、对白、因果链、在对的地方停下、用「写完检查一件事」自检。
   - 其他来源：2-5 句场景片段。
2. **准备其余章节**：按 `modules/page-spec.md` 各章节的写作规范生成核心机制、现实锚点、适用边界
3. **确定元数据**：domain（从固定词汇表选）、source、tags（discipline/ → apply/）
4. **准备 `scholar_refs`**：核心机制中引用的所有学者或思想家。每条包含 `full_cn`（中文全名）、`en`（英文原名）、`year`（代表年份或论文年份）、`key_claim`（归因于该学者的核心论断，供搜索核验）
5. 读取 `modules/write-page.md`，按**模式 A**执行写入、自检、lint、同步

多个概念逐个处理，每个写完后输出确认，再处理下一个。

### Step 6 — 可选：发起圆桌

所有概念页写入完成后，从本次新建的概念中找出张力最强的一个，询问是否发起圆桌讨论：

> 要不要就「{概念名}」发起圆桌讨论？我想到一个问题：
> **{问题}**
> 或者你来出题。

问题生成规则：必须是开放性问题，指向这个原理的核心张力或实践困境，不超过 2 句话。

用户确认后读取 `modules/roundtable.md`，以该问题为议题执行圆桌流程；否定则流程结束。

### Step 7 — 圆桌内容归位

圆桌流程执行完毕后，主动提示：

> 圆桌结束了，要把内容归位到概念页吗？

用户确认后，读取 `modules/write-page.md`，按**模式 B**执行圆桌沉淀追加、自检、lint、同步。

## 概念页规范

写入操作由 `modules/write-page.md` 统一处理。本模块只负责内容生成（提取概念、准备章节），不直接写文件。
