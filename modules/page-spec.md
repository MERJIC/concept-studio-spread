# 概念页规范

概念页格式规范的唯一权威。各模块引用本文件，不在模块内维护副本。

---

## frontmatter 模板（字段顺序和名称固定，不得增减或改名）

```yaml
---
name: 概念名称（English Name）
domain: [经济学]
date: YYYY-MM-DD
source: 阅读沉淀
tags: [discipline/行为经济学, apply/商业]
---
```

**source 字段**（5个，不得自创）：

| source 值 | 含义 |
|-----------|------|
| `寓言故事` | 从寓言故事模块产生 |
| `概念跳跃` | 从概念跳跃产生 |
| `对话整理` | 从日常对话中提炼 |
| `阅读沉淀` | 从阅读中提炼 |
| `圆桌讨论` | 从圆桌讨论中产生 |

**禁用字段（出现即错）：** `title` `slug` `created` `related` `updated` `aliases` `name_en` `status` `discipline`

---

## domain 固定词汇表

只能从以下 11 个顶层学科选，不能自创或填子学科名：

```
哲学 / 心理学 / 经济学 / 社会学 / 传播学 / 管理学 / 生物学 / 物理学 / 人类学 / 政治学 / 艺术
```

**规则：**
- 支持数组多值，跨学科概念填 `domain: [哲学, 艺术]`。单学科填单值即可
- 填起源学科，不填应用学科
- 母子学科不重复（行为经济学已隐含经济学）
- 场景（自媒体、职场等）一律用 `apply/` 标签，不进 domain

## 标签词汇表

**discipline/** — 概念的起源子学科，必填一个。可自行添加新子学科，但写完后必须告知用户：

```
哲学：伦理学 / 行动哲学 / 认识论 / 心灵哲学 / 形而上学 / 语言哲学 / 科学哲学 / 政治哲学 / 逻辑学 / 美学 / 中式哲学 / 批判理论 / 技术哲学 / 存在主义 / 现象学 / 精神分析
心理学：认知心理学 / 社会心理学 / 动机心理学 / 认知科学 / 教育心理学 / 发展心理学 / 临床心理学 / 人格心理学
经济学：行为经济学 / 制度经济学 / 信息经济学 / 金融学
社会学：组织社会学 / 文化社会学 / 社会学
管理学：组织行为学 / 知识管理 / 系统思维
生物学：行为生物学 / 演化生物学 / 控制论
物理学：量子物理 / 热力学 / 复杂系统 / 统计物理
传播学：传播学
社会学补充：流行病学
人类学：文化社会学
政治学：政治哲学 / 国际关系
艺术：美学 / 音乐理论 / 视觉理论 / 叙事学 / 文学理论
```

**person/** — 概念相关的学者/思想家，可多选。仅标记正文中有实质性引用的学者，tag 中提到的不算。**准入门槛：该学者在全库中出现频次 ≥5 次才可使用此标签。** 低于门槛的学者不标注，即使正文引用了。**标签值必须使用中文（学者的中文全名或短名），禁止使用纯英文、英文+下划线（如 Teece、Johnson_Clay）等格式。** 白名单（2025-05-25 锁定）：

```
维特根斯坦 / 康德 / 庄子 / 海德格尔 / 弗洛伊德 / 卡尼曼 / 布迪厄 / 福柯 / 胡塞尔 / 阿伦特
```

新人入围需经全库统计确认达到 ≥5 次门槛，更新白名单并告知用户。

**apply/** — 最容易落地的场景，可多选（共 10 个，不得自行添加）：

- `apply/自我` — 自我认知、动机、心理
- `apply/关系` — 人际、信任、群体
- `apply/制度` — 规则、组织、社会结构
- `apply/创作` — 审美、叙事、创作过程
- `apply/自媒体` — 平台运营、内容创作、流量机制
- `apply/商业` — 商业决策、市场行为
- `apply/组织` — 团队管理、组织结构、协作
- `apply/决策` — 判断、权衡、选择
- `apply/领导` — 权力、影响力、方向感
- `apply/教育` — 学习、教学、认知发展、学校与培训场景

## 正文结构

**必有（四个 section，顺序固定）：**

```markdown
## 核心机制

原理是什么，100-200字。可包含理论背景和核心争议。概念跳跃来源的，跳跃路径和关键判断写在这里（作为理解脉络），用 [[]] 链接起点概念。

## 入口场景

所有来源统一写成故事体——有名字的人在做具体的事，概念在这个场景中被碰到。必须包含三要素：①一个有名字的人（不是"某人""一个人"）②一个具体的情境（时间、地点、动作）③概念的第一次出现——人物被卡住、被惊到、或看到了之前没看到的东西。

纯抽象描述（如"从概念跳跃出发，起点是XXX""在讨论XX时引出"）不合格。

**所有 source 统一用完整叙事体（300-500字），写法以 `modules/parable.md` → Step 2 — 构建故事 为唯一叙事权威（典故优先、体裁、角色、把事情讲清楚、停在对的地方、写完检查）。**

不同 source 的差异仅在于：
- `寓言故事` / `概念跳跃`：必须严格走 parable.md 全流程
- `对话整理` / `阅读沉淀` / `圆桌讨论`：入口场景可不隐藏概念名，其余规则与寓言一致

## 现实锚点

这个原理在现实中最常见于哪种场景，1-3个具体例子。使用 bullet point 格式：`- **关键词**：描述`

## 适用边界

什么情况下这个原理不成立，或容易被误用。
```

**可选（有则写，没有不强求）：**

```markdown
## 圆桌沉淀

圆桌讨论后写入。完整原文保留、触发问题原文保留、出现的概念用 `[[]]` 标注。
```

**禁止出现的章节名：**
- 寓言
- 关联概念
- 相关概念
- 衍生问题
- 延伸
- 拓展
- 与XX的关系
- 与XX的区别

## 关联方式

正文中自然提到其他概念时，直接写 `[[概念名]]`。不需要刻意搜索关联，也不需要 ls 目录。没有自然出现的 `[[]]` 说明关联本来就不强。

---

## 学者名标注

正文中提到学者/思想家本人时，首次出现统一为「全名（英文原名）」，后续只用「姓氏/名字」。

- 首次：全名+英文名，如 `米歇尔·福柯（Michel Foucault）` 或 `庄子（Zhuangzi）`（无全名者只用名字）
- 后续：只用姓氏/名字，如 `福柯`、`庄子`

**首次出现的位置**：学者在正文中第一次被提及的位置（可能是核心机制、入口场景、适用边界、圆桌嘉宾表格等任何正文段落），后续指同一页内之后的所有引用。圆桌嘉宾表格中英文名列已含英文名，算作有效首次标注，表格之后的发言不再重复标注英文名。Noosphere 插件的 `checkScholarNames` 同时识别行内 `名字（English Name）` 格式和表格分列格式。

学者对照表见 `modules/scholar-dict.json`（Noosphere 插件自动读取，207 条）。新增学者时写入此 JSON 并告知用户。

**不加英文括注的情况：**
- 派生词/形容词：`康德式优美`、`福柯式的规训社会`、`马基雅维利者`（不加括注）
- `[[]]` 概念链接内部：`[[马基雅维利主义]]`（链接里的人名是概念名的一部分，不是指学者本人）
- 《》书名中的人名：`《庄子·齐物论》`（书名引用不算指学者本人）

## 圆桌沉淀格式规范

### 板块结构（顺序固定）

```markdown
## 圆桌沉淀

**日期：** YYYY-MM-DD
**触发问题：** {问题原文}

**嘉宾：**

| 人物 | 英文名 | 身份/领域 | 核心立场 |
|------|--------|----------|---------|
| 中文名 | English Name | 身份 | 一句话概括为何出现在这场讨论 |

### 第N轮：{争议焦点}

{各嘉宾发言}

### 主持人综述

{综述正文}

### 留存洞见

- {每条一句}
```

**必选 section**：元数据行（日期 + 触发问题 + 嘉宾）、轮次发言、主持人综述、留存洞见。四者缺一不可。无圆桌则不写此板块。

### 嘉宾行格式

用 markdown 表格，四列：

| 列 | 内容 | 要求 |
|----|------|------|
| 人物 | 中文名（全名） | 如有多音字用常用读法 |
| 英文名 | English Name | 括号内罗马拼写 |
| 身份/领域 | 身份或学科领域 | 如「哲学家」「进化生物学家」「行为经济学家」 |
| 核心立场 | 一句话概括 | 概括此人为何出现在这场讨论——他/她对这个概念的核心观点或独特角度 |

- 不用斜杠分隔的单行格式
- 核心立场栏应与该嘉宾在圆桌中的实际发言内容一致
- 部分古代人物无传统「英文名」，用拼音罗马化
- **禁止**标注 MBTI 人格类型

### 发言格式

```markdown
【人名】【行动标签】：发言内容

**简言之**：一句话压缩
```

- **人名**只用姓氏或常用名（嘉宾行已标注全名和英文名，此处不重复）
- **行动标签**固定 6 个：`陈述` / `质疑` / `补充` / `反驳` / `修正` / `综合`，不自行添加
- 发言人标签**不加粗体**——粗体会干扰阅读流，正文重点是内容本身
- `**简言之**：` — 冒号在粗体外
- 发言正文中提到其他学者时，遵循通用学者名标注规则（首次全名+英文，后续只用姓氏）

### 主持人综述

- 标题统一为 `### 主持人综述`（markdown 三级标题），不用粗体标题，不加轮次编号
- 多轮圆桌中每轮综述紧跟在对应轮次发言之后，各用独立的 `### 主持人综述`
- 内容为纯文字段落：提炼核心争议脉络 + 指出悬而未决的问题

### 图表

使用 ` ```text ` 代码块绘制 ASCII 结构图，不使用 ` ```mermaid ` 标记。Obsidian 对纯文本代码块原生渲染无兼容问题。

```markdown
```text
        A ──── B ──── C
        │             │
        D             E
```
```

图表类型选择：
- 坐标/光谱 → 横向排列，用 `←——→` 表示关系轴
- 层级/因果 → 纵向排列，用 `↓` 表示方向
- 矩阵 → 用 `───` 对齐列，行间用空行分隔

### 留存洞见

- 标题统一为 `### 留存洞见`
- 格式为 bullet list（`- `），每条洞见一句
- 不用编号列表，不用纯段落

## 自检清单

写完概念页后，逐条检查以下规则，全部通过才能交付。

1. **frontmatter 字段名合规**：所有字段名在模板列表内（name / domain / date / source / tags），无禁用字段
2. **name 字段格式**：`中文名（English Name）`，括号是中文全角括号，英文首字母大写
3. **tags 两类必填且在词汇表内 + person 可选**：`tags` 数组包含 `discipline/`、`apply/` 两类前缀，值全在上方词汇表内。`person/` 为可选附加层，仅当正文实质性引用的学者在白名单内时才标注（白名单及准入门槛见上方 person 说明）。**person/ 标签值必须为中文，禁止纯英文或下划线格式**。禁止单独使用 `discipline:` 字段——所有 discipline 必须放进 `tags` 数组
   - **tag 顺序固定为：discipline → apply → person**。同类别内部多个标签按需排列，不强制字母序。lint 会检测并自动修复顺序错误
4. **domain 合规**：数组格式，学科名在 11 个顶层固定值内
5. **正文无 h1 标题**：文件名即标题，正文从 `## h2` 开始
6. **章节结构完整且顺序正确**：核心机制 → 入口场景 → 现实锚点 → 适用边界 →（圆桌沉淀，可选）。无非标准章节名
7. **现实锚点使用 bullet point 格式**：`- **标题**: 内容`，不使用编号列表，不使用纯段落
8. **概念链接用 `[[]]` 自然嵌入**：正文自然出现时写 `[[]]`，不设独立的「相关概念」章节
9. **学者名标注合规**：指学者本人时首次出现 `全名（英文名）`（对照 `scholar-dict.json`），后续只用姓氏/名字；派生词和 `[[]]` 链接内部不加英文括注；Noosphere 插件自动检查
10. **圆桌嘉宾行格式合规**：嘉宾用四列表格（人物/英文名/身份与领域/核心立场）；每位嘉宾四列均需填写；禁止标注 MBTI
11. **圆桌沉淀格式合规**（如有此 section）：板块结构完整（元数据 → 轮次发言 → 主持人综述 → 留存洞见）；发言 `【人名】【标签】：` 不加粗体；`**简言之**：` 冒号在粗体外；主持人综述和留存洞见用 `### ` 标题；图表用 ` ```text ` 代码块，不用 ` ```mermaid `
12. **中文引号合规**：正文使用「」，禁止使用""（弯引号）
13. **禁止否定排比**：正文不使用「不是A而是B」「不仅A更B」句式，直接说B
14. **建页后强制跑 lint**：新建或修改概念页后，必须运行 `python3 scripts/lint_concepts.py --file 概念名` 通过质检，发现 F07/F10/F11 问题需当场修复
15. **`lint --fix` 安全边界**（批处理必读）：
   - 默认只报告：`python3 scripts/lint_concepts.py`（不写回）
   - `python3 scripts/lint_concepts.py --fix` **自动修复不含 F09**（学者短名替换已拆出）
   - 学者标注自动改字必须显式加 `--fix-scholars`；禁止对整库连跑两次 `--fix`/`--fix-scholars` 叠修
