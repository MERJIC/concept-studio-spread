# 概念跳跃 (Concept Hop)

以概念库中的**已有概念**为跳板，每次向外跳到一个**库里没有的新概念**。你感兴趣就深入，腻了就继续跳，想沉淀就调摄入模块。

---

## 起点选择（三种，开始时先确认）

**准备**：读取 `{ROOT}/memory/concept_lite.json`。
- `names` 数组 = 库内所有概念中文名，用于判断「是否已在库中」（O(1) 查找）
- `orphan_nodes` = 孤立概念列表（用于起点 C）
- `apply_index` = 按应用场景分组的概念列表
- `name_aliases` = 同义名映射（查重时也要检查）
- `name_en_index` = 英文名倒排索引（英文名查重用）

> **首次使用？** 如果 `{ROOT}/memory/concept_lite.json` 不存在，说明概念库尚未初始化。提示用户先执行一次「摄入」来创建初始索引。

后续所有「是否已在库中」的判断都基于这份已读取的索引。

**A · 随机抽取**（用户说「随便」「跳一跳」「给我新的」）
- 从索引的 `names` 数组中随机选一个已有概念作为起点

**B · 用户指定**（用户直接说一个概念名）
- 在索引 `names` 数组中搜索该概念名（同时检查 `name_aliases`），存在则作为起点
- 不存在则提示：「库里还没有这个，要不换一个，或者先建页？」

**C · 从孤立概念出发**（用户说「从孤立节点」「补链接」）
- 从索引的 `orphan_nodes` 中读取孤立概念列表（`fully_isolated` = 无入链无出链，`semi_isolated` = 有入链无出链）
- 孤立列表为空时回退到起点 A（随机抽取），并告知用户「当前无孤立概念，改为随机起点」
- 优先选 `semi_isolated`（已有入链说明有概念指向它，更容易找到延伸方向）

## 每一跳的格式

### 1. 起点简报（2-3 句）

用最压缩的语言说清这个已有概念的核心机制。不是解释给新人听，是给已经知道的人快速定位用的。

### 2. 跳向新概念（3 个，每次必须是库里没有的）

基于已读取的 concept_lite.json 内容确认以下推荐的概念都不在 `names` 中（同时检查 `name_aliases`）。

每个新概念给出：
- **概念名**（中文 + 英文原名）
- **一句话**：它是什么，和起点的关系是什么
- **来自哪个领域**
- **为什么有意思**：它解释了什么你可能没想到的现象

三个推荐按跳跃方式分类：
- **延伸跳**：同领域更深一层，直接相关
- **跨域跳**：换一个学科，但底层机制相似
- **意外跳**：乍看没关系，但有深层共鸣的

### 3. 问用户

```
跳哪个？（1/2/3）
或者你说一个你想跳的方向。
想沉淀当前的 → 说「沉淀这个」或「摄入」
继续下一跳 → 说「继续」
```

---

## 沉淀时补链接

用户说「沉淀这个」或「摄入」时，路由到 `modules/ingest.md` 执行摄入流程。ingest 建完页后（含索引刷新），**hop 负责补链接**：

1. 读取起点概念的 `.md` 文件（路径：`{ROOT}/概念页/{起点概念名}.md`）
2. 判断正文中是否自然可以加入新概念的 `[[]]` 链接
3. 如果可以，直接更新起点概念页，把链接补进去
4. 告知用户补了什么链接、加在哪里

**只补自然出现的链接，不强行加。**

补链接完成后，主动提示：
```
链接已补。继续跳？（说「继续」）
```

---

## Session 内记忆

记住本次已跳过的所有概念（起点 + 新概念），避免重复推荐。
用户说「今天到这」或「结束」时，输出本次跳跃路径：

```
今天的路径：
[起点概念] → [跳1] → [跳2] → ...

沉淀了：[已建页的] / 没有沉淀
```

不写入 memory，不做回忆测试，不强制总结。

---

## 原则

- 每次推荐的新概念**必须**是库里没有的，跳之前先检查
- 不复习旧概念，不做测试，不评分
- 用户腻了就跳，不解释、不挽留
- 推荐密度够就行，不要铺太长
