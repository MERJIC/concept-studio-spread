# 概念跳跃 (Concept Hop)

以概念库中的**已有概念**为跳板，每次向外跳到一个**库里没有的新概念**。你感兴趣就深入，腻了就继续跳，想沉淀就调 `/concept-ingest`。

概念库路径：`{ROOT}/概念页/`

---

## 起点选择（三种，开始时先确认）

**准备**：不需要预加载任何索引文件。所有查询通过命令按需执行：

```bash
# 查概念是否存在 / 查重
python3 scripts/sync_db.py -d "候选名"

# 随机起点（从库里随机取一个）
python3 scripts/sync_db.py --query "SELECT name FROM concepts ORDER BY RANDOM() LIMIT 1"

# 孤立概念列表（用于起点 C）
python3 scripts/sync_db.py --preset orphans

# 用户指定的概念是否在库中
python3 scripts/sync_db.py --query "SELECT id FROM concepts WHERE name = '概念名'"
```

**禁止读取** concept_lite.json、INDEX.md、concept_graph.json 或任何索引文件来获取概念清单或做查重判断。

**A · 随机抽取**（用户说「随便」「跳一跳」「给我新的」）
- 用 `--query "SELECT name FROM concepts ORDER BY RANDOM() LIMIT 1"` 随机取一个作为起点

**B · 用户指定**（用户直接说一个概念名）
- 用 `--query "SELECT id FROM concepts WHERE name = '概念名'"` 检查是否存在
- 存在则作为起点；不存在则提示：「库里还没有这个，要不换一个，或者先建页？」

**C · 从孤立概念出发**（用户说「从孤立节点」「补链接」）
- 用 `--preset orphans` 获取孤立概念列表
- 孤立列表为空时回退到起点 A（随机抽取），并告知用户「当前无孤立概念，改为随机起点」
- 优先选半孤立（有入链无出链）的概念

## 每一跳的格式

### 1. 起点简报（2-3 句）
用最压缩的语言说清这个已有概念的核心机制。不是解释给新人听，是给已经知道的人快速定位用的。

### 2. 跳向新概念（3 个，每次必须是库里没有的）

用查重命令确认每个候选不在库中（**3 个候选一次性批量提交**，一条命令）：

```bash
python3 scripts/sync_db.py -d "候选1名" "英文名1" "候选2名" "英文名2" "候选3名" "英文名3"
```
输出「✅ 可用」即可使用。不要读索引文件来手动比对。

每个新概念给出：
- **概念名**（中文 + 英文原名）
- **一句话**：它是什么，和起点的关系是什么
- **来自哪个领域**
- **为什么有意思**：它解释了什么你可能没想到的现象

三个推荐按跳跃方式分类：
- **延伸跳**：同领域更深一层，直接相关
- **跨域跳**：换一个学科，但底层机制相似
- **意外跳**：乍看没关系，但有深层共鸣的

### 3. 问用户

```
跳哪个？（1/2/3）
或者你说一个你想跳的方向。
想沉淀当前的 → 说「沉淀这个」或「摄入」
继续下一跳 → 说「继续」
```

## 沉淀时补链接

用户说「沉淀这个」或「摄入」时，concept-studio 路由到 `modules/ingest.md` 执行摄入流程。

**入口场景 = 按寓言写故事（必守）：** 跳跃沉淀的「入口场景」与寓言故事**同一套写法**，不是梗概、不是会议纪要。

建页前**必须先读取** `modules/parable.md` → **Step 2 — 构建故事**（第零步「典故优先」→ 第一步「找到那件事」→ 第二步「选择体裁」→ 第三步「让角色有血有肉」→ 第四步「把事情讲清楚」→ 第五步「在对的地方停下来」→ 「写完检查一件事」），逐步执行。

与寓言的差异仅一点：入口场景**可以**出现概念名（读者已在建页语境）；其余规则一致——有动作、有对白、让事情说话、结尾不总结感悟，篇幅约 300–500 字。`ingest.md` Step 5 同样适用此引用。

ingest 建完页后（含索引刷新），**hop 负责补链接**：

1. 读取起点概念的 `.md` 文件（路径：`{ROOT}/概念页/{起点概念名}.md`）
2. 判断正文中是否自然可以加入新概念的 `[[]]` 链接
3. 如果可以，直接更新起点概念页，把链接补进去
4. 告知用户补了什么链接、加在哪里

**只补自然出现的链接，不强行加。**

4. **运行 `python3 scripts/sync_db.py --file {起点概念名}`** — 补链接后必须同步（捕获新加的 [[link]] 关系）

补链接完成后，主动提示：
```
链接已补。继续跳？（说「继续」）
```

## Session 内记忆

记住本次已跳过的所有概念（起点 + 新概念），避免重复推荐。
用户说「今天到这」或「结束」时，输出本次跳跃路径：

```
今天的路径：
[起点概念] → [跳1] → [跳2] → ...

沉淀了：[已建页的] / 没有沉淀
```

不写入 memory，不做回忆测试，不强制总结。

---

## 原则

- 每次推荐的新概念**必须**是库里没有的，跳之前先检查
- 不复习旧概念，不做测试，不评分
- 用户腻了就跳，不解释、不挽留
- 推荐密度够就行，不要铺太长
